Quick Split

The developer tool for fast A/B tests

Register an account

Download the CLI (requires Python3.7+ and pip installed on your system).

$ pip install --user quicksplit

Register a new account.

$ quicksplit register

You'll be prompted for an email address and password. (We promise never to send you unhelpful marketing emails).

Create an experiment.

$ quicksplit create landing-page-test

You'll use the name of the experiment in the next steps, where you'll log exposure and conversion events for the experiment.

But before you can log events, you'll need to locate the public, staging token for your account.

$ export TOKEN=$(quicksplit tokens --staging --public)

Public tokens provide for limited access to your account, only allowing logging events. They should always be used in client-side applications such as a JavaScript web app.

Staging tokens are used to separate staging data from production data. They should be used in your development environment.


Log an exposure event

Log an exposure for your experiment by sending JSON encoded data to the /exposures route.

$ curl https://api.quicksplit.io/exposures \
  -H "Authorization: ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
        "experiment": "landing-page-test",
        "subject": 12345,
        "cohort": "control"
      }'

Exposures have three required fields: experiment, subject, and cohort.

  • experiment is the name of the experiment you created previously. In this case, it's landing-page-test
  • subject is typically a user id provided by your own backend.
  • cohort corresponds to which version of your application the user is experiencing. In an A/B test, you might use the labels "experimental" and "control" for your cohorts.


Log a conversion event

Log a conversion for your experiment by sending JSON encoded data to the /conversions route.

$ curl https://api.quicksplit.io/conversions \
  -H "Authorization: ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
        "experiment": "landing-page-test",
        "subject": 12345
      }'

Confirm that the exposure and conversion logging was successful by checking the recent events.

$ quicksplit recent --staging

┌────────────┬───────────────────┬─────────┬─────────┬───────┬───────────────────────────────┐
│ Type       │ Experiment        │ Subject │ Cohort  │ Value │ Last seen                     │
├────────────┼───────────────────┼─────────┼─────────┼───────┼───────────────────────────────┤
│ conversion │ landing-page-test │ 12345   │ control │       │ Tue, 25 Feb 2020 21:37:52 GMT │
│ exposure   │ landing-page-test │ 12345   │ control │       │ Tue, 25 Feb 2020 21:34:35 GMT │
└────────────┴───────────────────┴─────────┴─────────┴───────┴───────────────────────────────┘

Note that we used the special --staging flag to retrieve recent staging events. Omitting that flag would pull recent production events instead.


Download the results of your experiment

You can retrieve the statistical significance of your experiment as soon as it has started receiving events.

$ quicksplit results landing-page-test --staging

┌─────────┬──────────┬─────────────────┬────────────────────┐
│ Cohort  │ Subjects │ Conversion rate │ 95% conf. interval │
├─────────┼──────────┼─────────────────┼────────────────────┤
│ control │ 1        │ 1               │ [,]                │
└─────────┴──────────┴─────────────────┴────────────────────┘
Your test is not statistically significant. Either you need to collect more data, or there is no difference in means between the cohorts

Note that we're using the --staging flag here again because we've been using the staging token.

Frequently asked questions


Q: What sample sizes can I use for a single experiment?

You can log up to 3,000 subjects in a single experiment. In typical situations, that's a large enough sample size to detect ~1% differences in conversion rates.

Q: Can I run multiple experiments at the same time?

You can run up to three concurrent experiments on a single account. To stop an existing experiment, use the command quicksplit stop. To restart an experiment, use the command quicksplit start.
